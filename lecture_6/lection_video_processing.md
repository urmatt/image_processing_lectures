## 1. Введение: как устроено видео?

Видео — это не что иное, как **последовательность статических изображений** (кадров), сменяющих друг друга с определенной скоростью. Эта скорость называется **FPS** (Frames Per Second — кадры в секунду). 

Если мы умеем обрабатывать одно изображение (как мы научились в прошлых лекциях — переводить в ЧБ, применять фильтры, искать границы), то мы можем применять те же самые алгоритмы **к каждому кадру видеопотока**. Это и есть покадровая обработка.

```text
Видео = Кадр 1 -> Кадр 2 -> Кадр 3 -> ...
Обработка = f(Кадр 1) -> f(Кадр 2) -> f(Кадр 3) -> ...
```

## 2. Работа с веб-камерой в OpenCV

В библиотеке `OpenCV` для захвата видео с веб-камеры или из файла используется класс `cv2.VideoCapture`. 

### Основные методы VideoCapture:
- `cap = cv2.VideoCapture(0)` — открывает доступ к веб-камере (0 — индекс камеры по умолчанию).
- `ret, frame = cap.read()` — считывает следующий кадр. 
  - `ret` (boolean) — успешно ли считан кадр.
  - `frame` (numpy array) — сам кадр в виде матрицы пикселей.
- `cap.release()` — освобождает камеру, когда она больше не нужна.

### Пример 1: Трансляция с веб-камеры

В этом примере мы просто захватываем кадры и выводим их на экран, пока пользователь не нажмет клавишу `q`.

```python
# Файл: code/01_webcam.py
import cv2

def main():
    # Инициализация захвата видео (0 - первая доступная камера)
    cap = cv2.VideoCapture(0)

    # Проверка, что камера успешно открылась
    if not cap.isOpened():
        print("Ошибка: Не удалось открыть камеру")
        return

    print("Камера успешно запущена. Нажмите 'q' для выхода.")

    # Бесконечный цикл покадрового чтения
    while True:
        # read() возвращает флаг успешности и сам кадр
        ret, frame = cap.read()

        # Если кадр не получен, значит видео закончилось или произошел сбой
        if not ret:
            print("Не удалось получить кадр")
            break

        # Отразим кадр по горизонтали, чтобы он выглядел как в зеркале (опционально)
        frame = cv2.flip(frame, 1)

        # Вывод текущего кадра в окно
        cv2.imshow('Webcam Live', frame)

        # waitKey(1) ждет 1 миллисекунду нажатия клавиши.
        # Если нажата 'q', выходим из цикла
        if cv2.waitKey(1) & 0xFF == ord('q'):
            break

    # Обязательно освобождаем ресурсы
    cap.release()
    cv2.destroyAllWindows()

if __name__ == '__main__':
    main()
```

## 3. Детекция лиц в реальном времени

Для поиска лиц на кадрах мы можем использовать классический метод машинного обучения — **каскады Хаара (Haar Cascades)**. Этот метод был придуман в 2001 году и до сих пор отлично справляется с быстрым поиском лиц в реальном времени даже на слабых компьютерах.

В `OpenCV` уже встроены предобученные модели каскадов Хаара. Нам достаточно загрузить `haarcascade_frontalface_default.xml` и применить метод `detectMultiScale` к каждому кадру.

**Важно:** Каскады Хаара работают с **черно-белыми изображениями**, поэтому перед детекцией мы должны переводить кадр в градации серого.

### Пример 2: Детекция лиц

В этом примере мы будем захватывать кадр, находить на нем лицо и рисовать вокруг него зеленый прямоугольник.

```python
# Файл: code/02_face_detection.py
import cv2

def main():
    # Загружаем предобученную модель для поиска лиц из встроенных файлов OpenCV
    face_cascade = cv2.CascadeClassifier(cv2.data.haarcascades + 'haarcascade_frontalface_default.xml')

    cap = cv2.VideoCapture(0)

    if not cap.isOpened():
        print("Ошибка: Не удалось открыть камеру")
        return

    while True:
        ret, frame = cap.read()
        if not ret:
            break

        frame = cv2.flip(frame, 1) # Зеркальное отображение

        # Для каскадов Хаара нужно перевести изображение в ЧБ формат
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)

        # Детекция лиц
        # scaleFactor=1.1 - во сколько раз уменьшается размер изображения на каждом масштабе
        # minNeighbors=5 - сколько минимум соседей должно быть у кандидата, чтобы его признали лицом
        # minSize=(30, 30) - минимальный размер лица
        faces = face_cascade.detectMultiScale(gray, scaleFactor=1.1, minNeighbors=5, minSize=(30, 30))

        # Перебираем все найденные лица (x, y - верхний левый угол; w, h - ширина и высота)
        for (x, y, w, h) in faces:
            # Рисуем зеленый прямоугольник поверх оригинального (цветного) кадра
            cv2.rectangle(frame, (x, y), (x + w, y + h), (0, 255, 0), 2)
            
            # Добавим подпись
            cv2.putText(frame, 'Face', (x, y - 10), cv2.FONT_HERSHEY_SIMPLEX, 0.9, (0, 255, 0), 2)

        cv2.imshow('Face Detection', frame)

        if cv2.waitKey(1) & 0xFF == ord('q'):
            break

    cap.release()
    cv2.destroyAllWindows()

if __name__ == '__main__':
    main()
```

## 4. Практика: анонимизация (размытие) лиц

Теперь, когда мы умеем находить лица, давайте с ними «что-то сделаем». Одним из частых применений покадровой обработки является сокрытие (анонимизация) лиц, автомобильных номеров и конфиденциальной информации.

Для этого мы:
1. Найдем координаты лица (x, y, w, h).
2. Вырежем эту область из кадра: `face_roi = frame[y:y+h, x:x+w]`.
3. Применим к вырезанной области сильное Гауссово размытие.
4. Вставим размытую область обратно в исходный кадр.

### Пример 3: Блюр лиц (Face Blur)

```python
# Файл: code/03_face_blur.py
import cv2

def main():
    face_cascade = cv2.CascadeClassifier(cv2.data.haarcascades + 'haarcascade_frontalface_default.xml')
    cap = cv2.VideoCapture(0)

    if not cap.isOpened():
        print("Ошибка: Не удалось открыть камеру")
        return

    while True:
        ret, frame = cap.read()
        if not ret:
            break

        frame = cv2.flip(frame, 1)
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        
        # Находим лица
        faces = face_cascade.detectMultiScale(gray, scaleFactor=1.1, minNeighbors=5, minSize=(30, 30))

        for (x, y, w, h) in faces:
            # Вырезаем область лица (Region of Interest)
            # В numpy массивах сначала идет строка (Y), затем столбец (X)
            face_roi = frame[y:y+h, x:x+w]
            
            # Применяем сильное Гауссово размытие к лицу
            # Чем больше ядро размытия (99, 99), тем сильнее блюр
            blurred_face = cv2.GaussianBlur(face_roi, (99, 99), 30)
            
            # Вставляем размытое лицо обратно в оригинальный кадр
            frame[y:y+h, x:x+w] = blurred_face
            
            # Опционально: можно нарисовать рамку, чтобы было видно, где был блюр
            cv2.rectangle(frame, (x, y), (x + w, y + h), (255, 0, 0), 2)

        cv2.imshow('Anonymous Video', frame)

        if cv2.waitKey(1) & 0xFF == ord('q'):
            break

    cap.release()
    cv2.destroyAllWindows()

if __name__ == '__main__':
    main()
```

## 5. Самостоятельная работа

Попробуйте изменить код из `03_face_blur.py` (или написать свой скрипт `04_face_effect.py`) так, чтобы вместо размытия делать с лицом что-то другое. 

**Идеи:**
1. **Пикселизация (Цензура):** Вместо блюра, уменьшите размер `face_roi` до `10x10` пикселей, а затем снова увеличьте до исходного размера `w x h` с методом интерполяции `cv2.INTER_NEAREST`. Это создаст эффект пиксельной цензуры.
2. **Отрисовка очков:** Найдите в интернете картинку черных очков с прозрачным фоном (PNG) и попробуйте наложить её поверх найденного лица в верхней его половине.
3. **ЧБ фильтр только на лицо:** Оставьте весь кадр цветным, а само лицо делайте черно-белым.

## 6. Продвинутая детекция: Интеграция YOLO

Для поиска более сложных объектов, таких как бутылки, телефоны, люди или автомобили, каскады Хаара уже не подходят. Для этого применяются современные нейросети, например, **YOLO** (You Only Look Once).

В файле `code/04_task_integrate_yolo.py` показан пример использования YOLOv8 для поиска объектов в видеопотоке. 

### Практическое задание: Детекция пластиковой бутылки и её крышки

В качестве дополнительного задания посмотрите файл `code/05_task_bottle_detection.py`.
В нем реализована программа, которая решает следующую задачу:

1. Захватывает видео с веб-камеры и использует сеть YOLO для поиска бутылок (`bottle`).
2. **Анализ форм-фактора**: Сравнивает высоту и ширину найденной рамки (Aspect Ratio). Если бутылка вытянута (соотношение >= 1.5), прямоугольник становится зеленым. Иначе - оранжевым (возможно, бутылка обрезана в кадре).
3. **Поиск крышки**: Программа выделяет верхние 15% рамки бутылки, дополнительно центрируя область по ширине, чтобы показать, где находится "крышка".
